# 批量删除 UI 优化设计

> 日期：2026-07-05
> 主题：批量管理入口与退出交互优化

## 背景

当前「批量管理」功能存在三个体验问题：

1. **入口位置语义不符**：入口按钮放在顶部工具栏（与导出/导入并列），但批量删除是对"内容区"的操作，语义上更贴近筛选栏。
2. **命名含糊**：「批量管理」不够直白，用户不知道进去之后能做什么。
3. **「完成」按钮有歧义**：进入模式后操作栏的「完成」按钮，用户分不清是"确认删除"还是"退出模式"。

## 目标

- 入口移到筛选栏右侧，与筛选标签同行同高
- 文案改为「批量删除」，明确动作语义
- 退出路径清晰无歧义

## 设计

### 一、文案与命名

| 位置 | 原文案 | 新文案 |
|------|--------|--------|
| 顶部工具栏 | 「☑️ 批量管理」 | **移除** |
| 筛选栏右侧 | （无） | 新增「批量删除」入口 |
| 批量操作栏右端 | 「完成」 | 「✕ 取消」 |

### 二、布局

#### 普通状态（未进入批量模式）

```
[全部] [日记] [待办] [想法]                    [批量删除]
```

筛选栏（`.filters`）右侧贴一个入口按钮，与筛选标签同行同高（胶囊形），但用次要色调（描边、灰字），与"当前激活的筛选标签"在视觉上区分。

#### 批量模式（点击入口后）

```
[全部] [日记] [待办] [想法]               [批量删除]  ← 高亮激活态
─────────────────────────────────────────────────────────
[☐ 全选]   已选 0 条          [🗑️ 删除选中]  [✕ 取消]
```

- 入口按钮变高亮（主色填充），再点一次 = 退出
- 下方独立出现批量操作栏（沿用现有 `.batch-bar`，仅改文案）

### 三、交互

| 操作 | 行为 |
|------|------|
| 点「批量删除」入口（普通态） | 进入批量模式 |
| 点「批量删除」入口（高亮态） | 退出批量模式 |
| 点「✕ 取消」 | 退出批量模式 |
| 按 ESC 键 | 退出批量模式 |
| 点「🗑️ 删除选中」 | 二次确认后删除，并自动退出批量模式 |
| 拖拽框选 / 点卡片 / 全选 | 维持现有逻辑不变 |

**多退出途径不冲突**：入口高亮再点、取消按钮、ESC 三者都调用同一个 `exitBatchMode()`，用户用哪个都行，不会引起逻辑分歧。

### 四、代码改动点

| 文件 | 改动 |
|------|------|
| `index.html` | 删顶栏批量按钮；筛选栏加 `.filter-bar-actions` 容器放入口；`.batch-bar` 文案改「✕ 取消」 |
| `css/style.css` | 新增 `.batch-entry` 样式 + 高亮态 `.active`；调整筛选栏 flex 布局让入口右贴；清理顶栏 `.tool-btn` 中批量按钮相关 |
| `js/app.js` | 入口按钮事件改为 toggle（进/退同一按钮）；加 ESC 监听；`exitBatchMode` 由取消按钮/入口/ESC 共用 |

**不动**：框选逻辑、卡片选择逻辑、`store.deleteMany`、`timeline.applySelectionRect` 等。

## 非目标

- 不改框选交互（上一轮已完成）
- 不改导入导出
- 不增加"批量移动分类""批量改颜色"等其他批量操作（YAGNI）

## 验证清单

- [ ] 入口按钮在筛选栏右侧，与标签同行同高
- [ ] 进入批量模式后入口变高亮，再点退出
- [ ] 「✕ 取消」文案清晰，退出模式不删除
- [ ] ESC 键可退出
- [ ] 删除选中后自动退出批量模式
- [ ] 拖拽框选、单选、全选行为不受影响
